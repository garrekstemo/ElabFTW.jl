Actions

State-changing actions on experiments and items — locking, pinning, RFC 3161 timestamping, and cryptographic signatures. Each wrapper sends PATCH /{entity_type}/{id} with an {"action": ...} payload and returns the updated entity record.

Lock and pin

Both lock and pin are toggles at the API level — calling them on an already-locked (or pinned) entity flips the state back. This matches the eLabFTW spec and the UI behavior.

ElabFTW.lock_experiment — Function
lock_experiment(id::Int) -> Dict

Toggle the lock state on an experiment. A locked experiment cannot be edited. Calling this on an already-locked experiment unlocks it.

Returns the updated experiment record.

Example

lock_experiment(42)    # locks
lock_experiment(42)    # unlocks
ElabFTW.lock_item — Function
lock_item(id::Int) -> Dict

Toggle the lock state on an item (resource). See lock_experiment.

ElabFTW.pin_experiment — Function
pin_experiment(id::Int) -> Dict

Toggle the pin state on an experiment. Pinning bookmarks the entity in the user's dashboard. Calling this on an already-pinned experiment unpins it.

Returns the updated experiment record.

ElabFTW.pin_item — Function
pin_item(id::Int) -> Dict

Toggle the pin state on an item (resource). See pin_experiment.

Timestamp (RFC 3161)

ElabFTW.timestamp_experiment — Function
timestamp_experiment(id::Int) -> Dict

RFC 3161 timestamp an experiment. Requires the eLabFTW instance to be configured with a trusted timestamping service. Marks the entity's timestamped=1 and records timestamped_at.

Returns the updated experiment record.

Example

stamped = timestamp_experiment(42)
println(stamped["timestamped_at"])
ElabFTW.timestamp_item — Function
timestamp_item(id::Int) -> Dict

RFC 3161 timestamp an item (resource). See timestamp_experiment.

Requires the eLabFTW instance to have a trusted timestamping service configured. If it's not configured, the server returns HTTP 500 (surfaced as ServerError after retry exhaustion).

Sign

ElabFTW.sign_experiment — Function
sign_experiment(id::Int; passphrase, meaning) -> Dict

Cryptographically sign an experiment with the user's signing key.

Arguments

  • passphrase::String — the passphrase for the caller's signing key
  • meaning — an Int (10, 20, 30, 40, 50) or Symbol (:approval, :authorship, :responsibility, :review, :safety). See SIGN_MEANING.

Requires the eLabFTW instance to be configured with signing keys. If keys are not configured, the server returns HTTP 500 (this surfaces as a ServerError after retry exhaustion).

Example

sign_experiment(42; passphrase="secret", meaning=:approval)

Throws

  • ArgumentError — meaning is not in SIGN_MEANING (bad Int or unknown Symbol).
  • ServerError — signing keys not configured or passphrase wrong. The server returns HTTP 500 rather than 4xx for these; after retry exhaustion this surfaces as ServerError.
ElabFTW.sign_item — Function
sign_item(id::Int; passphrase, meaning) -> Dict

Cryptographically sign an item (resource). See sign_experiment.

ElabFTW.SIGN_MEANING — Constant
SIGN_MEANING :: Dict{Symbol, Int}

Mapping of signature-meaning symbols to the integer codes the eLabFTW API requires. Used by sign_experiment and sign_item.

SymbolCode
:approval10
:authorship20
:responsibility30
:review40
:safety50

Signing needs server-side signing keys. If they aren't configured, or the passphrase is wrong, the server responds with HTTP 500 (not 400) — this is a known eLabFTW quirk. Watch for ServerError on failure.

Accepts either an Int (10/20/30/40/50) or a Symbol (:approval, :authorship, :responsibility, :review, :safety) for meaning.
